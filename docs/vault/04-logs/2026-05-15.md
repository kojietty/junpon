# 2026-05-15 4キー転向 MVP

## ゴール

JUNGLE TAP のメカニクスを **左右 2 タップ → S/D/J/K の 4 キー**へ拡張する。判定ラインではなく **既存の達磨落とし式 (中央 1 本スタック)** を維持し、不正解は即ゲームオーバーではなく **スコア -1 ペナルティ** に変更。60 秒の制限時間でセッションが終わる。

## 決定事項 (ADR)

| 項目 | 決定 | 理由 | 代替案 |
|------|------|------|--------|
| 入力キー | S, D, J, K の 4 キー | 両手のホームポジション付近で 4 種類を打ち分けやすい | A/S, K/L など / 矢印キー |
| メカニクス | 中央 1 本スタックを維持 (達磨落とし式) | 既存 `CharacterStack` がそのまま使え、最小変更で 4 タイプ化できる | 4 レーン並列落下 / 判定ライン式 |
| ノーツタイプ数 | 4 (S/D/J/K に対応) | キー数と一致、視認性も維持できる | 2 (簡単) / 6+ (難) |
| 不正解の扱い | スコア -1 ペナルティ (床 0)、コンボリセット、軽い赤フラッシュ | 「下に来てから素早く押す」反射ゲーで即終了は厳しすぎる。スコア競争型に転換 | 即ゲームオーバー / ライフ 3 制 |
| 取り逃し (Miss) | 本仕様には存在しない (スタック式のため時間制限による Miss は発生しない) | 達磨落とし式は時間制限がないが、代わりに 60 秒タイマーで全体を区切る | スタック overflow で終了 |
| セッション長 | 60 秒固定 (`SESSION_DURATION_MS = 60_000`) | ハイスコア追求型に必要な明確な終端 | 90 秒 / 30 秒 / エンドレス |
| MVP スコープ | キーボードのみ・固定速度 (難易度カーブと pointer は Phase 2) | 「動く最小を早く」原則に従う | 全部入り v1 |
| 旧資産 | `Character.ts` / `CharacterStack.ts` を拡張流用 (削除しない) | 達磨落とし式維持により 4 タイプ拡張だけで足りる | 物理削除 + 新規 |

## 行動メモ

- `GameConfig.ts` の `COLORS.left/right` を `COLORS.lane[0..3]` に置換
- `CharacterSide` 型を `LaneKey = 0|1|2|3` に変更、`side` プロパティを `lane` にリネーム
- 旧 `gameOver()` の即終了は削除、代わりに `ScoreManager.fail()` で減算
- 60 秒タイマーは `this.time.delayedCall(SESSION_DURATION_MS, ...)` + 残り時間表示
- 不正解時は `cameras.flash(120, 255, 80, 80)` の軽いフラッシュのみ
- DifficultyCurve は今回触らず、Phase 2 で復活させる
